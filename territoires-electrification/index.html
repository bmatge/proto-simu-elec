100 territoires d'électrification — carte interactive

Les territoires (communes, EPCI, départements pilotes) volontaires pour accélérer l'électrification de leurs usages (transports, bâtiments, ménages). Sélection progressive d'ici l'été 2026, avec accompagnement État et engagements chiffrés à 2030.

 Prototype — données simulées

La DGEC publiera la liste réelle des 100 territoires d'ici l'été 2026. En attendant, ce prototype simule l'API avec un fichier JSON statique /data/territoires.json (100 entrées plausibles, distribution réaliste par région). Quand la vraie liste sortira, il suffira de remplacer le contenu du JSON — le pipeline dsfr-data reste inchangé. Voir la section Méthodologie & sources.

Méthodologie & sources de données

Ce prototype simule un endpoint API avec un fichier JSON statique servi à /territoires-electrification/data/territoires.json. Côté composants, c'est strictement la même chose qu'une vraie API : <dsfr-data-source url="..."> fetch le JSON, le distribue aux facettes, à la carte, aux KPIs via le bus data-bridge.

Pour passer en production il suffirait de remplacer le contenu du JSON (ou pointer url vers un endpoint DGEC). Aucun changement sur les composants.

Limites du prototype

  • Les coordonnées sont approchées (centroids communaux/EPCI publics + bruit ±0,05°).
  • Les engagements chiffrés sont générés proportionnellement à la population (0,8-2,5 PDC/1000 hab pour les bornes, 8-18 PAC/1000 hab, etc.) — réaliste pour un POC mais à confronter aux conventions réelles.
  • Le statut et la progression suivent une distribution réaliste mi-2026 (41 candidats, 37 retenus, 16 conventionnés, 6 en œuvre).
  • Les financements sont des montants plausibles (2-40 €/hab) — un mix réel impliquerait croiser plusieurs sources (Ademe AAP DECARB FLASH, ANAH MPR, Programme Advenir…).

Reproduction

Le générateur de données est versionné dans scripts/generate-data.py à côté de cette page. python3 scripts/generate-data.py re-génère le JSON avec une graine aléatoire fixe (reproductible). Pour faire évoluer le schéma, modifier le générateur et relancer.
